URLs

URL - Uniform Resource Locator

Example of an URL

http://www.udacity.com/path
Ports
Port is a always present in the request and allows to target different parts of the host.
Default port is ":80" and might not be specified in URL.

http://localhost:8000/

Query parameters (GET parameters)
A GET parameter always comes after the path and has a following structure name=value
Multiple GET parameters in a single URL are separated by "&" sign name=value&name1=value1

http://example.com/foo?p=1&q=neat

Fragments
Fragments are used to reference a particular part of the page we are looking at
Fragments are not being sent to the server along with the request and exist only in the browser
Fragments follow the query parameters and come last

http://example.com/foo?p=1&q=neat#fragment

HTTP requests and responses

HTTP - the main protocol of the web and stands for
HyperText Transfer Protocol
HTTP Requests
When we make a GET request to the server, request line follows the URL

http://example.com/foo

GET /foo HTTP/1.1
method path version
A typical HTTP request contains Headers. They have the following structure name:value
Two mandatory headers are Host and User-Agent

Host: www.example.com
User-Agent: chrome v.75

HTTP Responses
HTTP response is what the requesting party gets in a response to the request. In case of a successful request the server returns a file.
The most common response to our example request of GET /foo HTTP/1.1 would be HTTP/1.1 200 OK

The most commonly used status codes

200 OK Success
302 Found The file is on the server but located somewhere else
404 Not found There's no such document under specified path
500 Server error The server broke trying to handle your request
For more info on HTTP Status Codes

Typical resonse headers look like this:
HTTP/1.1 200 OK
Date: Sun Jul 21 2019 15:14:55
Server: Apache/2.2.3
Content-Type: text/html (or image/png, or other)
Content-Length: 1539

Servers

The main purpose of a server on the Internet is to respond to HTTP requests by senting content.
There are 2 types of content:
1) Static - pre-written file
2) Dynamic - files generated on the fly
